Các khái niệm

Directory:a folder used for storing multiple files - Directory là nơi chứa các file.

Repository:a directory where Git has been initialize to start version controlling our files - Repository là một thư mục nơi Git quản lý version các file của chúng ta.

untracked:newly created files but aren't tracked by Git yet

unstaged:files with change that have not been prepared to be committed

staged:files are already to be committed

deleted:files have been deleted and is waiting to be removed from Git

Staging area:files are not in our repository yet, we could add or remove files from the stage before we store them in the repository

Commit:a snapshot of our repository

GitHub:a remote server

local repository:repositories on our computer

remote repository:repositories on remote server, each remote repo has it's name and URL. Our typical repo's name is "origin"

HEAD: the Head is a pointer that holds your position within all our different commits. By default, HEAD points to our most recent commit.

Các câu lệnh cơ bản

$ git init- initialize a Git repository

$ git status- check current state of our project

$ git add file-name.txt- add file to the staging area, and Git start tracking changes made to this/these file(s)

$ git commit -m "Add filename.txt file"- store our staged changes, changes to be committed

$ git log --summarysee information for each commit

$ git remote add origin https://github.com/try-git/try_git.git- add a remote repo, this command takes a remote name and a repo URL. Our typical repo's name is "origin"

$ git push -u origin master- push our local changes to our origin repo (on GitHub), "origin" is the name of our remote, "master" is the default local branch name, -u tell Git to remember the parameters, so that next time we can simply run $ git pushand Git will know what todo

$ git pull origin master- check for changes on our GitHub repository and pull down any new changes

$ git stash- sometimes we go to pull, we may have changes we don't want to commit, use this command

$ git stash apply- re-apply our changes after pull

$ git diff HEAD- see different from our last commit

$ git diff --staged- see the changes we just staged

$ git reset octofamily/octodog.txt- unstage files, remove files have been added

$ git checkout -- octocat.txt- change files back to how they were at the last commit
